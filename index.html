📋 Overview

This community kit was built to help Microsoft cloud engineers and partners stand up a complete Entra Suite demo environment — with no guesswork. Each demo has its own Technical Runbook that walks you through every portal click, plus PowerShell scripts to automate the repetitive setup and reset tasks.

Demo 1

Access Reviews

Periodic review of group membership with sign-in recommendations, one-click "Accept All", and auto-applied results.

Demo 2

Lifecycle Workflows

Automated joiner / leaver identity lifecycle — TAP delivery, license assignment, account disable — all policy-driven.

Demo 3

Verified ID + Face Check

Passwordless onboarding with biometric Face Check — new-hire access package and contractor visitor pass.

Demo 4

Internet Access (GSA)

Web content filtering via Global Secure Access — block categories, security profiles, and Conditional Access enforcement.

Demo 5

Private Access (GSA)

No-VPN access to internal web apps, SMB file shares, and RDP — using the Private Network Connector.

🚀 Quick Start

1

Download and unzip the kit

Click Download Kit above. Unzip to a local folder — e.g. C:\EntraSuiteDemo\

2

Read the Series Overview

Open 00-Overview\Entra-Suite-Demo-Series-Overview.docx — it explains how all five demos connect end-to-end.

3

Install the Graph module (once per machine)

Navigate to Demo2-LifecycleWorkflows\Scripts\ and run:

PowerShell 
# One-time setup — run once per machine
.\01-Install-GraphModule.ps1
4

Customise the placeholder values

Every script that needs tenant-specific values has a ← CHANGE comment. See the Customise for Your Tenant section below.

5

Open a demo's Technical Runbook and follow it

Each demo folder has a *-Technical-Runbook.docx — follow it top to bottom for the portal configuration steps.

⚙️ Customise for Your Tenant

All scripts and documents use generic placeholders. Replace them before running:

PlaceholderReplace withWhere
yourtenant.onmicrosoft.com Your demo tenant domain All scripts, README, runbooks
admin@yourtenant.onmicrosoft.com Your Global Admin UPN $AdminUpn and $AssignUserUpn variables at top of each script
[Your Org Name] Your organisation name Verified ID issuer label in the portal + script output text
<YOUR-SALES-ONBOARDING-PACKAGE-ID> GUID of your Sales Onboarding access package Demo 3 reset/remove scripts — find in Entra admin center → Identity Governance → Access packages
<YOUR-CONTRACTOR-PASS-PACKAGE-ID> GUID of your Contractor Pass access package Same as above
ℹ️
Finding access package GUIDs Entra admin center → Identity Governance → Entitlement management → Access packages → open the package → copy the ID from the URL bar.

🔍 Demo 1 — Access Reviews

Demo 1

Access Reviews — Identity Governance

Periodic review of group membership with AI-driven recommendations and auto-apply

No scripts needed

What This Demo Covers

A quarterly access review of the Demo-FinanceAdmins group. The platform generates approve/deny recommendations based on the last 30 days of sign-in activity. Reviewers accept all recommendations in one click. Auto-apply removes denied users from the group automatically.

Files in This Demo

📄
Access-Reviews-Technical-Runbook.docx
Step-by-step portal configuration and demo walkthrough with click paths

Setup Steps

1

Create the demo group

In Entra admin center → Groups → create Demo-FinanceAdmins (security group). Add demo users: Carol Newhire, Dave Leaver, Stale Demouser, Manager Demo.

2

Create an access review

Identity governance → Access reviews → New access review → review the Demo-FinanceAdmins group. Set reviewer to Manager Demo. Enable recommendations, auto-apply, and justification required.

3

Wait 24–48 hours

The recommendation engine needs sign-in history to populate. Create the review the day before the demo.

⚠️
Important — create the review at least 24 hours before the demo The sign-in recommendation engine needs time to analyse activity data. A review created minutes before the demo will show no recommendations.

2

Configure a web content filtering policy in Global Secure Access that blocks a specific category (e.g. gambling, social media). Show the block page on a Windows 11 VM that has the GSA client installed. Then optionally show the admin-side policy and the CA policy delivering it.

Files in This Demo

🔒 Demo 5 — Private Access (GSA)

Demo 5

Private Access — No-VPN Internal Resources

Access internal web apps, SMB file shares, and RDP through the Private Network Connector — no VPN

10 scripts included

Architecture

VMRolePrivate IP
gsa-appsrvResource server — hosts the internal web app (IIS), SMB share, and RDP192.168.50.4
gsa-connectorPrivate Network Connector — bridges the resource server to Microsoft's cloud10.0.0.5
Client VMEntra-joined Windows 11 with GSA client — the end-user test machinedifferent subnet
⚠️
Critical — resource server must NOT be in the same subnet as the client If both VMs share the same /24 subnet, Private Access silently refuses to tunnel. The resource server must be in a different IP range from the client VM. The scripts default to 192.168.50.4 for this reason.

First-Time Setup

PowerShell / BAT
# ── On gsa-appsrv (RDP in and run) ──────────────────────────────
Setup-Resources-ONECLICK.bat    # installs IIS + SMB share + enables RDP

# ── On your admin machine (after connector is registered) ────────
# Edit $AssignUserUpn and $ResourceIP at the top of the script first
.\Build-Demo5-Config.ps1         # creates IP segments + assigns user

Soft Reset (Before Each Demo)

PowerShell
# Removes segments + assignment. VMs and connector stay running.
.\Reset-Demo5-Soft.ps1

# Rebuild the config
.\Build-Demo5-Config.ps1

Connector Troubleshooting

See Scripts\README-Commands-Reference.md for full diagnostics. Quick checks:

PowerShell (run on connector VM)
# Are connector services running?
Get-Service WAPCSvc, WAPCUpdaterSvc | Format-Table Name, Status -Auto

# Can the connector reach the resource?
"Web 80:  " + (Test-NetConnection 192.168.50.4 -Port 80   -WarningAction SilentlyContinue).TcpTestSucceeded
"SMB 445: " + (Test-NetConnection 192.168.50.4 -Port 445  -WarningAction SilentlyContinue).TcpTestSucceeded
"RDP 3389:" + (Test-NetConnection 192.168.50.4 -Port 3389 -WarningAction SilentlyContinue).TcpTestSucceeded
